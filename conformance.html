4.7 - Embedded Content

Chrome, Firefox, and IE all displayed resized images differently. Resized images in IE look harsh a crispy, while resized images in Chrome and Firefox look smooth. Resized images in Firefox look a bit more blurry than in Chrome.

All three browsers displays the alt text when there is an image with a bad src. Chrome displays a rectangle with the alt text inside. Firefox only displays the alt text. IE displays a tiny rectangle with an "X" in the middle and then the alt text to the right of the tiny rectangle.

All three browsers displays iframes similarly with slight color differences. The most noticeable difference is the different scrollbar used by the iframe in all three browsers.

The math element does not work on Chrome or IE, but Firefox is able to display the mathematical notation described by the math element.

Shapes created by the svg element are displayed the same way in Chrome and Firefox. IE displays shapes least elegantly. Borders around the shapes are thicker than in Chrome and Firefox. In IE, it is also possible for shapes to overlap surrounding elements or text.

The embed element works in Chrome, but Firefox and IE does not appear to be displaying media files embedded with the embed element. It is possible that the issue is due to a missing plugin.

An HTML page embedded using the object element look similar in Chrome, Firefox, IE with the only difference being the scrollbar.

Videos and sound embedded with the video and audio elements work in all three browsers. The difference is that all three have different controls.

The track element does not appear to be working in Chrome. It correctly displays subtitles in embedded media in Firefox and IE, but not in Chrome.
